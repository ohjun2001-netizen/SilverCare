SilverCare 프로젝트 피드백 및 개발 계획
1장. 프로젝트 개요 및 현재 방향
프로젝트 주제
SilverCare는 고령자를 대상으로 한 VR 기반 실버케어 미니게임 플랫폼입니다. 어르신들이 익숙하게 느낄 수 있는 바둑, 고스톱, 카드 맞추기, 골프 등의 활동을 VR 환경에서 제공하여 인지 활동, 여가 활동, 정서적 안정감을 지원하는 것을 목표로 합니다.
개발 목표
고령자가 부담 없이 접근할 수 있는 친숙한 콘텐츠 제공
기억력, 판단력, 순서 예측, 규칙 이해 등 인지 자극 요소 포함
VR 환경에서 손쉽게 선택하고 조작할 수 있는 인터랙션 구성
텍스트 안내와 음성 안내를 통해 사용자가 현재 해야 할 행동을 명확히 이해하도록 지원
복지관이나 교육 현장에서 짧은 시간 안에 체험 가능한 시연형 콘텐츠 완성
고령자 특성을 고려한 직관적이고 안전한 UX 설계
현재 개발 방향의 변화
초기에는 다양한 콘텐츠를 많이 구현하는 데 초점을 두었지만, 중간 피드백 과정에서 기능 수가 많아도 전체 완성도가 낮아 보일 수 있다는 의견을 받았습니다. 이에 따라 기능 확장보다 핵심 콘텐츠 완성도를 높이는 방향으로 전략을 수정했습니다. 현재는 퀴즈와 노래 맞추기 기능은 제외하고, 바둑, 고스톱, 카드 맞추기, 골프 중심으로 시연 가능한 완성도를 높이고 있습니다. 특히 고령자 대상 프로젝트라는 점을 고려하여 다음 요소를 우선 개선 대상으로 보고 있습니다. UI 가독성, 버튼 크기, 한글 안내 및 음성 안내, 간단한 조작 흐름, VR 오조작 방지 UX 중점으로 남은 기간 개발하려고 합니다.
현재 남길 핵심 콘텐츠
바둑 사활문제: 좌표 선택, 정답 판정, 힌트 및 TTS 안내
바둑 복기: JSON 기보 데이터를 기반으로 수순 재생
고스톱: 화투패 이미지 기반 카드 진행, 주요 룰과 점수 계산
카드 맞추기: 기억력 기반 카드 매칭 게임
골프: 물리 기반 공 이동, 코스 선택, 타수 계산
제외 또는 후순위 콘텐츠
퀴즈/노래 맞추기
2장. 현재까지의 구현 현황
메인 로비
현재 로비는 각 콘텐츠로 이동할 수 있는 중심 화면 역할을 합니다. 게임 버튼을 통해 바둑 사활문제, 바둑 복기, 다음 수 맞히기, 카드 맞추기, 고스톱, 골프 등으로 이동할 수 있습니다. 최근에는 메뉴 명칭을 정리하고, 완성도가 낮은 기능은 숨기거나 후순위로 돌리는 방향을 검토하고 있습니다. 로비 UI는 고령자가 쉽게 이해할 수 있도록 버튼 크기 확대, 한글 안내 문구 정리, 게임별 짧은 설명 추가가 필요한 상태입니다. 앞으로 로비는 단순한 게임 목록이 아니라 “인지 활동”, “기억력 활동”, “여가 활동”처럼 목적을 함께 보여주는 화면으로 개선할 계획입니다.
바둑 콘텐츠
바둑 콘텐츠는 JSON 데이터 기반으로 구현되어 있습니다. baduk_problems.json에는 문제 설명, 난이도, 흑돌/백돌 좌표, 정답 좌표, 힌트, TTS 문구가 포함되어 있습니다. Unity에서는 JSON 데이터를 로드하여 바둑판 위에 자동으로 돌을 배치하고, 사용자가 좌표를 선택하면 정답 여부를 판정합니다. 또한 힌트 시스템과 TTS 안내를 통해 사용자가 문제를 이해할 수 있도록 구성되어 있습니다. 바둑 복기 기능은 기보 데이터를 기반으로 수순을 재생하는 방식으로 구현되어 있습니다. 다만 고령 사용자 관점에서는 19x19 격자 구조 특성상 원하는 좌표가 아닌 인접 좌표를 잘못 선택할 가능성이 있습니다. 현재는 좌표 선택 즉시 판정되는 구조이므로, 향후 다음과 같은 UX 개선을 검토하고 있습니다. * 선택 좌표 강조 표시 * 반투명 돌 미리보기 * 착수 전 확인 단계 * 마지막 수 취소 기능 등 구현 예정입니다.
카드 맞추기
카드 맞추기는 같은 그림의 카드 두 장을 찾는 기억력 게임입니다. 난이도에 따라 카드 개수가 달라지며, 랜덤 배치된 카드 중 같은 ID를 찾아 맞추는 구조입니다. 기본 로직은 구현되어 있으나 고령 사용자 특성을 고려하면 다음 개선이 필요합니다. * 카드 열람 시간 증가 * 선택 취소 UX 보완 * 한글 UI 정리 * 결과 안내 가독성 개선 등 을 계획중에 있습니다.
고스톱
고스톱은 화투패 48장을 월별 데이터로 정의하고 카드 이미지를 기반으로 구현했습니다. 게임 시작 시 손패, 바닥패, 더미를 구성하고, 같은 월의 패를 비교하여 진행됩니다. 고령 사용자 관점에서는 손패 선택 실수 가능성이 있어 다음 UX 개선이 필요합니다. * 선택 패 강조 표시 * 제출 전 확인 단계 * 고/스톱 UI 가독성 개선 등을 계획 중에 있습니다.
골프
골프는 Unity 물리 엔진 기반으로 구현되었습니다. 공 이동, 코스 선택, 타수 계산이 가능하며 VR 컨트롤러 움직임을 스윙 강도로 활용하는 방향으로 구성하고 있습니다. 고령 사용자의 조작 부담을 줄이기 위해 조작 안내와 결과 피드백을 단순하게 제공하는 방향으로 개선하고 있습니다.
3장. 진행 중 느낀 어려운 점 및 개발 이슈
1. 기능 수가 많아 완성도가 분산되는 문제
초기에는 여러 미니게임을 많이 보여주는 것이 프로젝트의 장점이라고 생각했습니다. 하지만 실제 피드백을 받아보니, 기능 수가 많아도 각 기능의 UI와 조작 흐름이 다듬어지지 않으면 전체 프로젝트가 미완성처럼 보일 수 있다는 점을 확인했습니다. 이 문제를 해결하기 위해 퀴즈와 노래 맞추기는 최종 시연 범위에서 제외하고, 핵심 콘텐츠 중심으로 완성도를 높이는 방향으로 결정했습니다.
2. VR 오조작 문제
VR 환경에서는 손 떨림이나 컨트롤러 조작 정확도 문제로 의도하지 않은 선택이 발생할 수 있습니다. * 대응 방향: 선택 카드 등 강조 표시, 진행하기 전 확인 단계, 되돌리기 기능
4장. 향후 개발 계획 및 일정
개발 전략
1주차 (5/18 ~ 5/24) — 핵심 게임 기본 흐름 완성
최종 시연 범위 확정 및 팀원별 담당 파일 분배
로비 UI 정리 (버튼 크기 확대, 메뉴명 한글화, 게임 설명 문구 추가)
바둑 문제 로딩 / 정답 판정 / 힌트 / TTS 안내 흐름 확인
고스톱 UI 정리 (손패, 먹은 패, 점수 표시, 고/스톱 버튼)
카드 맞추기 한글화 및 골프 UI 문구 정리
로비 ↔ 각 게임 씬 전환 통합 흐름 테스트
2주차 (5/25 ~ 5/31) — 고령자 UX 개선 + 버그 수정
고령자 친화 UI 기준 적용 (글자/버튼 크기, 색 대비, 안내 문구 단순화)
오조작 방지 UX 개선 (바둑 되돌리기, 고스톱 패 선택 확인 단계, 카드 열람 시간 연장)
VR 인터랙션 안정화 (레이 선택 방식 점검, 손 떨림 오조작 방지 검토)
TTS 보강 및 시연 환경에서 서버 연결 안정성 확인
전반적인 버그 수정 (씬 전환 오류, UI 겹침, 입력 오류 등)
데모 시나리오 제작 및 1차 내부 리허설
3주차 (6/1 ~ 6/7) — 최종 완성도 + 발표 준비
리허설 피드백 반영 및 잔여 버그 수정
발표 자료 완성 (피드백 반영 내용, 진행 현황, 개발 이슈 정리)
시연 환경 최종 세팅 (VR 기기, TTS 서버, 씬 빌드 상태 점검)
데모 영상 촬영
예상 질문 답변 정리 및 최종 리허설
